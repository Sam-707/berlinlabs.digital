# MenuFlows Validation Scripts

**Objective:** Validate demand for MenuFlows QR menu system through direct restaurant outreach and landing page testing.

---

## Method 1: Direct Restaurant Outreach

### Target Locations

**Berlin Districts to Visit:**
1. **Mitte** (High tourist foot traffic)
   - Friedrichstraße, Oranienburger Straße, Hackescher Markt
   - Restaurant density: High
   - Best times: 10:00-11:30 (before lunch rush)

2. **Kreuzberg** (Trendy, younger owners)
   - Bergmannkiez, Schlesisches Tor
   - Restaurant density: Very high
   - Best times: 14:00-17:00 (after lunch, before dinner)

3. **Prenzlauer Berg** (Upscale, owner-operated)
   - Kollwitzplatz, Helmholtzplatz
   - Restaurant density: High
   - Best times: 10:00-11:30 or 15:00-17:00

**Target Restaurant Types (Priority Order):**
1. Independent cafes (often lack digital menus)
2. Small ethnic restaurants (often have outdated menus)
3. Owner-operated restaurants (decision maker on-site)
4. Trendy spots (more open to innovation)
5. Avoid: Large chains (need corporate approval)

## Outreach Scripts

### Script 1: Initial Approach (In-Person)

```
"Entschuldigung, sind Sie der Inhaber?"

Wait for: "Ja" or "Was kann ich für Sie tun?"

"Hi, ich bin [Dein Name]. Ich habe eine kurze Frage.
Ich entwickle ein einfaches QR-Menüsystem für Restaurants.

Haben Sie gerade ein digitales Menü oder drucken Sie noch?"
```

**If they have digital menu:**
```
"Ah, super! Wie funktioniert das? Sind Sie zufrieden damit?"

[Listen to response - useful market research]

"Würden Sie offen sein für eine Alternative, die einfacher und günstiger ist?"
```

**If they don't have digital menu:**
``"Genau das habe ich gedacht. Viele Restaurants drucken noch.

Ich baue ein einfaches System:
- Keine App-Installation nötig
- QR-Code, Gäste scannen einfach

- Sie können Preise und Gerichte in 2 Minuten ändern

Ich suche 5 Restaurants, die es 3 Monate lang kostenlos testen.
Keine Kreditkarte, keine Verpflichtung.

Wäre das interessant für Sie?"
```

**Expected Responses:**

**Option A: "Ja, das klingt gut"**
```
"Super! Hätten Sie jetzt 5 Minuten?
Ich zeige Ihnen, wie es funktioniert."

[Show demo on phone]

"Hät Sie nächste Woche Zeit für ein 10-minütiges Gespräch?
Ich zeige Ihnen alles und beantwortete alle Fragen."

[Get contact info or schedule follow-up]
```

**Option B: "Vielleicht" / Skeptical**
```
"Verstehe ich. Was wäre das größte Problem, das wir lösen müssten?"

[Listen to pain points]
[Address specific concerns]

"Wäre es möglich, dass ich nächste Woche vorbeikomme
und es Ihnen in 5 Minuten zeige? Wenn es Ihnen nicht gefällt,
kein Problem."

[Get contact info for follow-up]
```

**Option C: "Nein, danke" / Not interested**
```
"Alles klar, danke trotzdem!
Darf ich Ihnen eine Visitenkarte hinterlassen?
Falls Sie es später ausprobieren möchten."

[Leave card, move to next restaurant]
```

## Objection Handling

### Objection 1: "Unsere Gäste nutzen keine Handys"

**Response:**
```
"Das verstehe ich. Aber 80% der Deutschen haben ein Smartphone.
Gerade jüngere Gäste (bis 50) erwarten fast immer ein digitales Menü.

Außerdem:
- Keine Kontaktinfektion (noch relevant für manche)
- Keine verschmutzten Menükarten
- Immer aktuelle Preise und Gerichte

Können wir es eine Woche testen? Wenn es niemand nutzt,
können Sie es einfach wieder entfernen."
```

### Objection 2: "Das ist zu teuer"

**Response:**
```
"Verstehe. Lassen Sie uns zusammenrechnen:

Eine gedruckte Speisekarte:
- Design: 50-200 Euro
- Druck: 1-5 Euro pro Karte
- Aktualisierung: Immer neue Kosten

Bei MenuFlows:
- Einmaliger QR-Code-Druck: 5 Cent
- Unbegrenzte Aktualisierungen: Inklusive
- Design: Inklusive

Wenn Sie nur 2-3 Mal pro Jahr etwas ändern,
haben Sie es in einem Jahr raus.

Außerdem: 3 Monate kostenlos testen Sie es ja erstmal."
```

### Objection 3: "Das ist zu kompliziert"

**Response:**
```
"Das ist genau das Problem, das wir lösen!

Keine App-Installation.
Keine technische Kenntnisse nötig.
Wenn Sie E-Mail schreiben können, können Sie MenuFlows benutzen.

Ich richte Ihnen alles in 10 Minuten ein.
Das verspreche ich."

[Dann: sofortiger Demo, wenn möglich]
```

### Objection 4: "Ich habe keine Zeit"

**Response:**
```
"Verstehe! Deshalb richte ich Ihnen alles ein.

Sie müssen nur:
1. Mir Ihre aktuelle Speisekarte geben (per Foto oder E-Mail)
2. 5 Minuten für eine kurze Demo
3. Den QR-Code ausdrucken und aufstellen

Alles andere mache ich.

Wann hätten Sie 5 Minuten?"
```

### Objection 5: "Wir nutzen schon [Wolt/Lieferando/etc.]"

**Response:**
```
"Toll! Das bedeutet, Sie sind bereits digital.

Aber MenuFlows ist anders:
- Keine 30% Provision bei Bestellungen
- Gäste sehen Ihr ganzes Menü (nur Bestelloptionen)
- Gäste können direkt anrufen (für Reservierungen)
- Sie behalten die Kontrolle

MenuFlows ist für Gäste IM Restaurant, nicht für Lieferung.
Ergänzung, nicht Konkurrez.

Können wir es 3 Monate kostenlos testen?"
```
