# BerlinLabs Documentation

This folder contains internal documentation for the BerlinLabs product studio.
These files are **not** included in the production build (excluded in `tsconfig.json`).

---

## Files

### Studio Rules
- **`StudioRules.tsx`** - The complete studio constitution
  - Defines what we are (product studio vs. experiments)
  - Operational principles (friction first, validation gates)
  - What we avoid and decision rules
  - Renderable as a React component for internal reference

### Developer Documentation
- **`HANDOFF.md`** - Frontend developer handoff guide
  - Quick start commands
  - Folder structure
  - Integration seams (API, forms, routing)

### Marketing Strategy
- **`marketing-strategy.md`** - Overall marketing strategy
  - Vision and business model (PROOF → SHIP → ITERATE)
  - Product positioning (MenuFlows focus)
  - Target audiences and competitive landscape
  - Success metrics and KPIs

- **`linkedin-strategy.md`** - LinkedIn content strategy
  - Profile optimization (company page + founder profile)
  - Content pillars (3 posts/week schedule)
  - Engagement strategy and connection templates
  - Content calendar templates

---

## Usage

These are internal reference documents. The TSX files can be imported for viewing within the application if needed for internal tools, but are not part of the public-facing site.

**Marketing files are strategic references** for external communication planning.
